Chapter 1. System-Level Overview

AXI DMA 아키텍처 개요 및 역할 파악

Memory-Mapped 환경과 AXI-Stream 환경 간의 고속 데이터 전송 아키텍처를 체계적으로 분석합니다.

1.1 DMA의 정의 및 연결 구조

AXI DMA는 단순한 메모리 복사 블록을 넘어, CPU의 제어 명령(Configuration)을 기반으로 AXI4 메모리 버스트 트랜잭션과 AXI-Stream 데이터 파이프라인을 상호 연동하는 핵심 Bridge 인터페이스입니다. 본 문서에서는 Control(AXI-Lite), Memory Access(AXI4 Burst), Data Flow(AXI-Stream), Advanced Operation(Scatter-Gather)의 메커니즘을 단계적으로 심층 분석합니다.

1. AXI-Lite (Control Interface)

CPU가 DMA 내부 제어 레지스터(CSR)에 접근하기 위한 프로토콜입니다. Payload 데이터 처리가 불가하며 Source/Destination Address, Transfer Length, Interrupt Configuration 등 환경 설정 값을 전송합니다.

2. AXI4 Full (Memory Interface)

실질적인 대용량 데이터 전송 경로입니다. DMA 엔진이 AXI Master로 동작하여 System Memory(예: DDR RAM) 영역에 대한 고속 Burst Read/Write 트랜잭션을 주도합니다.

3. AXI-Stream (Data Pipeline)

Target 하드웨어 가속기(IP)와 직접 연결되는 고속 데이터 인터페이스. 암호화 코어, 필터(MAC), FFT 모듈과 같이 메모리 주소 체계 없이 데이터 순차 흐름 및 Packet Boundary 정합성이 요구되는 블록에 적용됩니다.

1.2 어떤 DMA IP를 선택해야 하는가?

AXI DMA라는 이름 때문에 모든 전송을 하나의 IP로 처리한다고 생각하기 쉽지만, 실제 설계에서는 데이터가 지나가는 형태에 따라 IP 선택이 달라집니다.

참고: Memory-to-Memory 전송을 원한다면?

만약 하드웨어 IP 없이 단순히 DDR RAM의 A 영역에서 B 영역으로 메모리를 복사하고 싶다면, AXI DMA(Stream 기반)가 아닌 AXI CDMA (Central DMA)라는 별도의 IP를 사용하는 것이 일반적입니다. CDMA는 스트림 포트 없이 AXI4 Full 인터페이스만으로 메모리 간 복사를 수행합니다.
