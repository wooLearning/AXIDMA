Chapter 2. The Power of AXI4 Full

데이터 고속도로: AXI4 Full Burst 트랜잭션 메커니즘

DMA가 대규모 데이터를 고속 전송할 수 있는 핵심 요인은 CPU의 제어 처리와 AXI4 Burst 데이터 경로를 물리적/논리적으로 완벽히 분리한 아키텍처에 기인합니다.

2.1 MMIO 실행 시퀀스

STEP 1

CONTROL 레지스터에서 Enable 및 Interrupt Enable 설정

DMA 구동 엔진을 활성화합니다. Interrupt Enable을 동시 설정할 경우, 전송 완료 시 CPU가 Polling 없이 비동기 알림(IRQ)을 수신할 수 있습니다.

STEP 2

SOURCE / DESTINATION 레지스터 기준 주소 할당

단순 Memory-to-Memory 전송 모델에서는 Source와 Destination 모두 System Memory 영역의 주소값을 갖습니다.

STEP 3

NUM 레지스터를 통한 트랜잭션 파라미터(시작, Chunk, 길이) 설정

본격적인 데이터 전송 트리거 시점입니다. Chunk는 연속 Burst의 단위 크기를 의미하며, Length/Bytes 파라미터는 총 전송 바이트 수를 정의합니다.

STEP 4

GO/BUSY 플래그 Polling 대기 또는 IRQ 수신 대기

펌웨어/드라이버 레벨에서 상태 레지스터의 GO/BUSY 비트 하강(완료)을 반복 확인하거나, 인터럽트 활용 시 Pending 상태와 IRQ 출력을 감지하여 완료를 판별합니다.

2.2 Burst 트랜잭션 설계 주요 항목

  • DMA 적용 목적: CPU 연산 자원 점유를 최소화하고, 단일 명령어 기반 메모리 접근 한계를 극복하여 대역폭 한계까지 Burst 효율을 극대화합니다.
  • 필수 제어 레지스터: Source Address, Destination Address, Transfer Length, Chunk Size, DMA Status (Done/Interrupt).
  • Chunk 파라미터의 의의: Chunk는 장주기 Burst를 분할 실행할 단위 크기를 지칭하며, 내부 AXI Data Width와 버퍼(FIFO) Depth 제약을 바탕으로 산출됩니다.
  • Address Alignment: Source와 Destination 간의 정렬 규칙(Alignment)이 부합해야 하며, Misaligned Start 발생 시 내부 FSM(Finite State Machine)이 최상단에서 보정 트랜잭션을 선행 처리합니다.
  • 주요 검증 요소: CSR 설정 시퀀스의 유효성 확인, 전송 상태 레지스터(Status) 관측, 그리고 Destination 메모리 영역 값 무결성 점검이 최우선 항목입니다.

2.3 CPU 제어 Sequence (Conceptual Architecture)

CPU(또는 BFM)가 DMA 프로세스를 구동하기 위해 요구되는 단계적 레지스터 접근 및 제어 흐름 모델입니다. 각 단계별 CSR 활성화 시점을 논리적으로 분류하였습니다.

01

Enable

DMA 동작 허용과 인터럽트 정책 설정

02

Source

읽기 시작 주소를 DMA에 전달

03

Destination

쓰기 시작 주소를 DMA에 전달

04

Start

전송 길이와 chunk를 담아 GO 트리거

05

Observe

GO 하강, DONE, IRQ로 완료 확인

핵심 규칙: 주소와 길이를 모두 준비한 뒤 마지막에 시작 트리거를 넣습니다. 완료 관찰은 polling 또는 interrupt 중 하나로 할 수 있고, interrupt pending은 별도 clear 단계가 필요합니다.

2.4 Driver/Firmware 관점의 제어 알고리즘

소프트웨어 스택에서의 실제 제어 코드는 하단 시퀀스로 요약될 수 있습니다. 핵심 원칙은 Destination 및 Source 주소를 선제적으로 매핑한 후, 최종적으로 GO 비트를 트리거하는 것입니다. 이후 GO 비트 하강 감지 또는 Interrupt Pending Flag 확인을 통해 트랜잭션의 정상 종료를 판별합니다.

CONTROL

EN/IE 초기화

DMA 모듈 활성화와 더불어, 필요 시 트랜잭션 완료 통보용 Interrupt를 허용합니다.

ADDRESS

DST / SRC 매핑

두 메모리 주소 간 하위 비트 Alignment 일치 여부를 사전 검증해야 합니다.

START

트리거 파라미터(GO+CHUNK+BYTES)

총 전송 길이와 분할 Burst(Chunk)를 레지스터에 동시 인가하여 기동 조건을 확립합니다.

WAIT / CLEAR

완료 검증 및 Interrupt Clear

트랜잭션 종결 후 발생한 Pending Interrupt는 소프트웨어 계층에서 명시적으로 Clear(1-to-Clear)해야 합니다.

Pseudo-code

enable DMA
write DST
write SRC
write GO/length
poll or IRQ clear

2.5 AXI-Lite와 AXI4 Full 인터페이스의 본질적 차이

AXI-Lite (Control Pathway)

1주소 : 1데이터 전송 (Single Access). CPU(Master)가 DMA 내부 설정용 레지스터(Slave CSR)에 단일 값을 쓰고 읽는 경로입니다.

AXI4 Full (Data Pathway)

1주소 : N데이터 연속 전송 (Burst Access). DMA(Master)가 System Memory(Slave)에 연속적인 데이터 페이로드를 요청하는 경로로, 1회의 주소 전달 시 최대 256 beat에 이르는 대용량 데이터를 처리합니다.

2.6 AXI4 Specification 기반 Burst 타이밍 해석

AXI4 프로토콜은 5개의 통신 채널이 상호 독립적인 Handshake를 수행합니다. 전 채널을 관통하는 최상위 원칙은 VALID 신호와 READY 신호가 동일한 ACLK Rising Edge에서 동시에 Assert(High)된 시점에만 유효한 트랜잭션이 성립한다는 것입니다. 다음 다이어그램은 AMBA 공식 사양의 타이밍 차트를 재구성한 것으로, DMA 하드웨어가 어떻게 다중 Read/Write 요청을 Overlap하여 처리 성능을 극대화하는지 설명합니다.

Read dependency

Master 측에서 주소 유효(ARVALID)를 선언하면 Slave 측이 수락(ARREADY)하여 연결이 수립됩니다. 이후 Slave가 반환(RVALID)하는 데이터 채널에서는 Master의 수락(RREADY) 시점에 구동되며 사이클이 소진(Beat 소비)됩니다.

Write dependency

Write 사이클은 주소(AW) Handshake와 데이터(W) Handshake 양측이 완전하게 충족되어야 정상 처리됩니다. Slave는 마지막 데이터 블록(WLAST) 수신 후 응답 채널(BVALID)을 통해 쓰기 완료 상태의 건전성을 통지합니다.

화살표 읽는 법: 단일 화살표는 끝 신호가 시작 신호 전/후 어느 쪽에서도 assert될 수 있음을 의미합니다. 이중 화살표는 끝 신호가 시작 신호 assert 이후에만 assert될 수 있는 필수 선행 조건입니다.

Overlapping Read Bursts

주소 burst A와 B가 먼저 겹쳐 들어가고, R 채널에서는 slave가 준비되는 순서대로 data beat와 RLAST를 반환합니다.

AR / R channel

Write Burst

AW 주소 채널과 W 데이터 채널은 별도로 handshake되며, 마지막 W beat의 WLAST 이후 B 채널에서 OKAY 응답이 반환됩니다.

AW / W / B channel

Read Burst Overlap

AR 채널을 통해 다중 주소 트랜잭션을 선행 인가(Pipelining)하면, R 채널은 Slave 준비 상태에 따라 시분할로 데이터를 반환하므로 Throughput 최적화가 가능합니다.

Write Response 조건

BVALID는 AW Handshake와 전체 W Data Beat의 Handshake가 완전히 종료된 이후에만 유효성을 가집니다. 패킷 종단은 WLAST 신호로 식별됩니다.

DMA 아키텍처 설계 포인트

Read 코어와 Write 코어를 분리하고 중간에 Asynchronous/Synchronous FIFO를 배치함으로써, Backpressure 상황에서도 양측 버스트 효율 저하를 방지할 수 있습니다.

2.7 AXI4 개별 채널 동적 타이밍 시뮬레이션

AMBA AXI4 사양은 구동 주소, 기록 데이터, 기록 응답, 독출 주소, 독출 데이터의 5개 채널을 완전 독립적으로 운영합니다. 고성능 DMA는 해당 특성을 활용하여 내부 Read/Write 파이프라인을 이원화합니다.
