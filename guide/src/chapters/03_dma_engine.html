Chapter 3. AMBA AXI-Stream Protocol

Address-Free Data Pipeline 아키텍처

AXI4 Full 사양이 메모리 주소 기반의 범용 트랜잭션을 규정한다면, AXI-Stream은 VALID/READY Handshake와 Packet Boundary 지시자만을 활용하여 대역폭 한계까지 직렬화된 데이터를 전송하는 단방향 고속 인터페이스입니다.

3.1 AXI-Stream 확장 이전의 코어 엔진 아키텍처

Read Core

Source Base Address를 기점으로 제한된 크기의 AR/R 채널 트랜잭션을 순차 생성합니다. Misaligned Access 발생 시, Boundary 맞춤 연산을 선행한 후 Aligned Bulk Burst 모드로 자동 전환됩니다.

Synchronous FIFO

Read Pipeline과 Write Pipeline 간의 시간적 역학(Temporal Dynamics) 차이를 보상하는 비동기/동기 버퍼입니다. 통신 응답 지연에 따른 Backpressure가 전체 시스템을 즉각 구속하지 않도록 완충 작용을 수행합니다.

Write Core

FIFO 버퍼에서 인출한 페이로드를 바탕으로 Destination 메모리에 대응하는 AW/W/B 채널 신호를 합성합니다. 시작 주소 정렬 여부를 판단하여 부분 기록(Partial Write)과 정규 기록단계를 스케줄링합니다.

구성요소 대표 상태 핵심 역할
Read Engine IDLE, UNALIGNED_ALIGN, ALIGNED_BURST, WAIT_RESP 전체 전송 요구량과 현재 주소의 Alignment 상태를 평가하여, 규격에 부합하는 ARLEN(Burst Length)/ARSIZE(Burst Size) 필드를 동적 생성합니다.
Sync FIFO Room 카운터 / Item 카운터 추적 Read 및 Write 도메인을 독립적으로 Decoupling하여 트래픽 병목에 의한 파이프라인 스톨(Stall) 파급을 국소화합니다.
Write Engine IDLE, UNALIGNED_ALIGN, ALIGNED_BURST, WAIT_RESP 버퍼로부터 유입된 Data 및 Strobe 마스크를 취합하여 Write Burst 사이클을 구동하며, 규정된 BVALID 응답까지 이력을 관리합니다.

개념도: 내부 엔진 협업 흐름 (Cooperative Data Flow)

Read Engine

Source 어드레스를 기점으로 AR/R 트랜잭션을 전개합니다. 하드웨어적 Alignment 불규칙성이 감지될 경우 단발성 트랜잭션으로 주소를 보정한 후 대규모 Aligned Burst를 수행합니다.

→

Sync FIFO

물리적 지연 요소(Latency) 불균형을 극복하기 위해 Read 채널에서 병합된 Data 및 Byte Strobe 신호계를 일시적으로 큐잉(Queueing) 처리합니다.

→

Write Engine

Destination 어드레스 기반으로 AW/W 채널을 전개합니다. Strobe 벡터를 해석하여 유효 바이트 범위만을 메모리에 커밋(Commit)하며 트랜잭션을 정상 종단합니다.

Activation 조건

CSR 컨트롤러가 GO 비트를 수락함과 동시에 내부 파이프라인이 Source 및 Destination 베이스 어드레스를 페치하여 구동 상태로 진입합니다.

Backpressure 제어

FIFO Full 상태 발생 시 Read 파이프라인의 Valid 인가가 중단되며, Empty 상태에서는 Write 파이프라인이 Stall하여 동적 균형을 유지합니다.

Termination 조건

Read 측의 마지막 펄스와 Write 측의 최종 확인 응답(B Channel)이 모두 충족되었을 때 FSM 프로세스가 종결 지점으로 회귀합니다.

RTL 아키텍처 관점 주요 상태 방정식

세부 하드웨어 논리를 관통하는 코어 제어 로직의 불리언(Boolean) 상관 관계를 추상화할 경우 다음과 같은 주요 방정식으로 도출됩니다.

Busy 인디케이터

`Read_Active || Write_Active` 시 논리적 Busy 상태가 유지됩니다.

Done 트리거

`Read_Complete && Write_Complete` 조건의 교집합에서 최종 상태 논리가 성립됩니다.

FIFO 임계 제어

Read는 `Room_Threshold > 0`일 때, Write는 `Item_Threshold > 0`일 때 동작이 인가됩니다.

Read Interface: AR Handshake 성립 이후 버스 상의 유효 R-Beat 데이터를 버퍼 공간에 이적합니다.
Write Interface: 버퍼 데이터 및 Byte Strobe를 디코딩하여 AW/W 트랜잭션으로 환산 및 메모리 반환 응답을 대기합니다.

3.3 AXI-Stream 핵심 신호선

Signal Source Engineering Description
TDATA Master 실제 페이로드(Payload) 데이터.
TVALID Master 현재 사이클의 TDATA/TKEEP/TLAST/TUSER가 유효함을 표시합니다. 데이터는 TVALID와 TREADY가 동시에 High인 클럭 엣지에서만 이동합니다.
TREADY Slave 수신측이 데이터를 받을 준비가 되었음을 표시합니다. TREADY가 0이면 송신측은 같은 데이터를 유지해야 하며, 이것이 AXI-Stream의 Backpressure입니다.
TLAST Master 논리 패킷의 종단을 지시하는 신호입니다. 송신 모듈(Tx)은 논리 단위(예: 영상의 1 라인, 블록 단위 연산 결과)의 마지막 버스 사이클에 맞추어 TLAST 펄스를 인가하며, DMA 하드웨어는 이를 감지해 데이터 수납 스코프 한 단위를 조기 종결시키고 레지스터 상태를 반영합니다.
TKEEP / TSTRB Master 마지막 beat처럼 버스 폭보다 실제 유효 바이트가 적을 때 어떤 byte lane이 유효한지 표시합니다. 패킷 길이가 데이터 폭의 배수가 아닐 때 특히 중요합니다.
TUSER Master 프로토콜 외 부가 의미를 전달하는 sideband입니다. AXI4-Stream Video에서는 일반적으로 프레임 시작(SOF)을 나타내는 데 사용됩니다.

3.4 Backpressure와 Deadlock 조건

AXI-Stream은 주소가 없기 때문에, 정상 동작 여부는 거의 전부 TVALID/TREADY/TLAST 조합에서 결정됩니다. 타겟 IP가 계산 중이라 TREADY를 내리면 DMA의 MM2S 출력도 멈추고, S2MM 쪽 버퍼가 꽉 차면 앞단 전체에 정지가 전파됩니다.

TREADY=0은 에러가 아니라 흐름 제어입니다.

다만 설계가 이 정지 상태를 견디지 못하고 데이터를 바꾸거나 TLAST를 잃으면 바로 프로토콜 위반입니다.

TLAST 누락은 S2MM 완료 불가로 이어집니다.

수신 DMA는 지정 길이 또는 패킷 종료 조건을 기다리므로, TLAST가 오지 않으면 인터럽트가 발생하지 않는 Hang처럼 보일 수 있습니다.

FIFO/Skid Buffer는 선택이 아니라 안전장치입니다.

연산 IP의 latency와 DDR burst 응답 변동을 흡수해, 양쪽의 순간적인 준비 상태 차이가 전체 deadlock으로 번지는 것을 줄입니다.

3.5 AXI4-Stream Video와 VDMA 관점

영상 스트림은 단순 byte stream보다 구조가 엄격합니다. 한 프레임은 여러 라인으로 구성되고, 각 라인 길이와 메모리 stride가 맞아야 하므로 일반 AXI DMA보다 AXI VDMA가 자연스러운 경우가 많습니다.

Video 개념 AXI4-Stream 표현 VDMA 설정과의 관계
SOF TUSER 새 프레임의 첫 픽셀/첫 beat를 표시합니다.
EOL TLAST 보통 한 라인의 마지막 beat를 의미합니다.
HSIZE / VSIZE 라인 바이트 수 / 라인 개수 프레임의 2D 크기를 지정합니다.
Stride 메모리 내 다음 라인 시작 간격 패딩이 있는 frame buffer에서 HSIZE보다 클 수 있습니다.
판단 기준: 데이터가 “패킷 몇 개” 수준이면 AXI DMA로 충분할 수 있습니다. 하지만 프레임 동기화, 라인 stride, genlock, display/capture 경로가 등장하면 VDMA로 보는 것이 안전합니다.
