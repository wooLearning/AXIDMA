Chapter 4. Advanced Execution Architecture

Scatter-Gather와 Linked List Descriptor 아키텍처

단편화된 물리 메모리 공간을 CPU의 Buffer Descriptor (BD) 조립 과정을 통해 단일 연속 트랜잭션으로 가상화하여 전송하는 병렬적 구조를 다룹니다.

4.1 핵심 오개념 불식: BD(Buffer Descriptor)의 생성 주체

가장 빈번한 오해 중 하나는 메모리 상의 BD가 사전 정의된 상수이거나 하드웨어가 스스로 구축한다는 가정입니다.

현실의 구현에서는 DMA 하드웨어 기동 직전 단계에서, 운영체제(OS) 최하단의 Device Driver가 시스템 동적 RAM 스페이스를 할당받아 인위적으로 BD 체인을 레코딩합니다. 즉, CPU가 주동자가 되어 물리 주소의 맵핑 포인터와 설정값들을 일일이 기입하여 Linked List를 완성한 뒤, 그 Entry Point (CURDESC)만 DMA 하드웨어에게 인가하는 방식입니다.

>_ CPU Device Driver - Pseudo C Flow

// 1. Descriptor 메모리 확보: CPU가 DMA와 공유 가능한 RAM 영역을 확보

BD* bd0 = dma_alloc_coherent(sizeof(BD), &bd0_dma);// CPU virtual pointer + DMA bus address

BD* bd1 = dma_alloc_coherent(sizeof(BD), &bd1_dma);

// 2. CPU가 descriptor field 작성: DMA가 이해하는 주소는 dma_addr_t

bd0->NXTDESC = bd1_dma;// 다음 descriptor의 DMA 주소

bd0->BUF_ADDR = block_A_dma;// 실제 데이터 버퍼의 DMA 주소

bd0->LENGTH = 4096;

bd0->CONTROL = SOF;// 패킷 시작 지시

bd1->NXTDESC = tail_dma;// 여기까지 처리하면 종료

bd1->BUF_ADDR = block_B_dma;

bd1->LENGTH = 2048;

bd1->CONTROL = EOF;// 패킷 마지막 지시

// 3. AXI-Lite로 DMA에게 시작 위치와 끝 위치를 통보

DMA_REG->CURDESC = bd0_dma;// DMA가 처음 읽을 descriptor 주소

DMA_REG->TAILDESC = tail_dma;// 마지막 주소를 쓰는 순간 DMA 시작

CPU pointer와 DMA address를 구분해야 합니다.

CPU가 descriptor를 쓰기 위해 사용하는 포인터와 DMA 하드웨어가 AXI 버스에 내보내는 주소는 다를 수 있습니다. Linux 드라이버에서는 보통 CPU virtual address와 dma_addr_t를 함께 관리합니다.

Descriptor도 캐시 동기화 대상입니다.

coherent 메모리가 아니면 BD field를 작성한 뒤 DMA가 보기 전에 clean/flush가 필요하고, DMA가 status를 업데이트한 뒤 CPU가 읽기 전 invalidate가 필요합니다.

4.2 CPU와 DMA의 비동기적 협업 모델 단계

현대적인 가상 메모리(MMU) 환경에서는 대용량 논리 버퍼가 단편화된 물리 페이지 스페이스로 분산 파편화될 확률이 높습니다. SG-DMA는 이러한 조각들을 Descriptor Chain으로 체계화하고, DMA 트랜잭션이 해당 리스트를 순차 지향하여 실제 페이로드 세그먼트들을 순서대로 일괄 병합 전송합니다.

Phase 1. Software (Kernel)

BD Chain Allocation

운영체제 드라이버는 동적 할당된 RAM 영역에 개별 BD 노드를 인스턴스화하고, 포인터(NXTDESC)로 각 노드를 연결하여 Ring 또는 Linear 형태의 커스텀 리스트를 구축합니다.

Phase 2. AXI-Lite API

SG Engine Trigger

제어 인터페이스(AXI-Lite)를 거쳐 SG DMA 내부의 Current Descriptor 레지스터와 Tail Descriptor 레지스터에 각각 시작 노드와 타겟 노드의 주소를 Push하면 즉각적인 하드웨어 패치가 개시됩니다.

Phase 3. AXI4 / AXI-Stream

Autonomous Execution

SG DMA는 자체적인 AXI4 Master 모듈을 활성화하여 BD를 로컬 SRAM에 Fetch-Decode한 후 데이터 청크를 복사하며, 처리가 완료된 BD에 대해서는 Status 필드를 Update합니다.

4.3 Buffer Descriptor 필드 맵

BD는 “코드 파일”이 아니라 CPU가 RAM에 써 놓는 작은 작업 카드입니다. DMA는 이 카드를 AXI4로 읽고, 실제 데이터 버퍼를 찾아가며, 마지막에 상태 칸을 다시 써 줍니다.
