Chapter 5. Memory Mapped I/O

Interactive CSR Memory Map

CPU-DMA 간의 비동기 제어 권한 이양을 위한 Memory-Mapped Register 구조와 Bit-Field 정의를 심층 조명합니다.

5.1 레지스터 요약

이름 Offset Access 의미
Control 제어 영역 RW DMA enable, interrupt enable, interrupt pending 상태 제어
Transfer Info 전송 정보 영역 RW/RO mix GO 시작 비트, BUSY/DONE 상태, CHUNK 크기, 총 BYTES 길이
Source 읽기 주소 영역 RW 읽기 시작 주소
Destination 쓰기 주소 영역 RW 쓰기 시작 주소

5.2 Control Register

CONTROL 레지스터는 하드웨어 코어의 Global 활성화 및 Interrupt 정책을 관장하며, Pending Interrupt의 명시적 초기화(W1C) 기능을 담당합니다.

31 EN
30 .. 2 RSVD
1 IP
0 IE
비트에 마우스를 올리면 의미가 표시됩니다.

5.3 Transfer Info Register

트랜잭션 실행 트리거 및 상태를 요약하는 복합 Register입니다. 펌웨어 로직은 본 영역을 기동 인가용으로 쓰거나 상태 검증용으로 읽습니다.

31 GO
30 BUSY
29 DONE
28 .. 24 RSVD
23 .. 16 CHUNK
15 .. 0 BYTES
비트에 마우스를 올리면 의미가 표시됩니다.

5.4 SOURCE/DEST Address Register

단순한 메모리 포인터 구조를 지니나, 하이엔드 트랜잭션에서는 이 주소의 하위 비트 Alignment 특성이 실질 성능 편향을 유도합니다.

레지스터 주소 설명
Source 읽기 주소 영역 Read Engine의 시작 주소 기준이 됩니다.
Destination 쓰기 주소 영역 Write Engine의 시작 주소 기준이 됩니다.

5.5 MMIO 시퀀스 체인 권장안

A. Initialization Phase

CONTROL CSR을 선행 조작하여 DMA 구동 허가 상태와 비동기 Interrupt 활성 범위를 확립합니다.

B. Address Mapping Phase

SOURCE 및 DEST 레지스터 쌍을 매핑합니다. 두 영역 간의 Alignment 대칭성이 Bus 효율 최적화를 보장합니다.

C. Execution Trigger Phase

NUM 컨트롤 구조체에 누적 Transfer Byte 포맷과 Burst Chunk 배분 지수를 동시 합성하여 GO 명령을 인가합니다.

D. Completion Verification Phase

BUSY 플래그 하강 또는 Interrupt Signal 인입을 근거로 파이프라인 무결성을 진단합니다.

위 모델링은 일반 프로세서 인터페이스 뿐 아니라 BFM 시뮬레이션 환경에서도 동일하게 통용되는 표준적 MMIO 패턴입니다. 소프트웨어 Layer는 제어의 타이밍을 지휘할 뿐, 물리적 전송 상태 전이 및 Alignment 분할은 하드웨어 코어가 자율 통제합니다.

실무 아키텍트 가이드

가변 CHUNK 길이는 AXI Databus 폭(Width)에 대한 직교배수로 설정함이 원칙입니다. 과도한 CHUNK 스펙의 오버 커밋(Over-commit)은 FIFO Depth Capacity와의 상관관계에 따라 시스템 Bus Deadlock을 초래할 수 있음에 유의하십시오.
