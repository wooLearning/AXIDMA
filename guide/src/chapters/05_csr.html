Chapter 5. Memory Mapped I/O

인터랙티브 CSR 레지스터 맵

CPU가 DMA를 제어하기 위해 접근하는 memory-mapped register를 역할과 비트 단위로 정리합니다.

5.1 레지스터 요약

이름 Offset Access 의미
Control 제어 영역 RW DMA enable, interrupt enable, interrupt pending 상태 제어
Transfer Info 전송 정보 영역 RW/RO mix GO 시작 비트, BUSY/DONE 상태, CHUNK 크기, 총 BYTES 길이
Source 읽기 주소 영역 RW 읽기 시작 주소
Destination 쓰기 주소 영역 RW 쓰기 시작 주소

5.2 Control Register

CONTROL은 DMA를 켜고 인터럽트 정책을 정합니다. pending interrupt는 여기서 clear합니다.

31 EN
30 .. 2 RSVD
1 IP
0 IE
비트에 마우스를 올리면 의미가 표시됩니다.

5.3 Transfer Info Register

NUM은 시작 트리거이자 상태 요약 레지스터입니다. 소프트웨어는 보통 이 값을 써서 시작하고 다시 읽어 완료를 확인합니다.

31 GO
30 BUSY
29 DONE
28 .. 24 RSVD
23 .. 16 CHUNK
15 .. 0 BYTES
비트에 마우스를 올리면 의미가 표시됩니다.

5.4 SOURCE / DEST 주소 레지스터

주소 레지스터 자체는 단순하지만, 내부 core는 이 주소들의 하위 정렬 비트를 매우 민감하게 사용합니다.

레지스터 주소 설명
Source 읽기 주소 영역 Read Engine의 시작 주소 기준이 됩니다.
Destination 쓰기 주소 영역 Write Engine의 시작 주소 기준이 됩니다.

5.5 공개용 제어 알고리즘

A. 준비 단계

CONTROL에서 DMA enable과 interrupt 정책을 먼저 결정합니다.

B. 주소 단계

SOURCE와 DEST에 메모리 시작 주소를 넣습니다. 두 주소의 alignment 관계가 전송 품질을 좌우합니다.

C. 시작 단계

NUM에 total bytes와 chunk 정보를 담고 GO 의미의 start trigger를 줍니다.

D. 관찰 단계

BUSY/DONE/GO 상태 또는 interrupt pending을 확인해 완료를 판단합니다.

이 흐름은 CPU 드라이버든 BFM 검증이든 공통으로 나타나는 MMIO 패턴입니다. 즉, CSR 맵은 단순하지만 상태 전이와 정렬 제약은 내부 제어기가 책임진다고 이해하면 됩니다.

실무 체크

CHUNK는 data bus width의 배수로 잡는 것이 안전합니다. 0으로 두면 data width 단위의 기본 묶음처럼 해석할 수 있고, 너무 큰 CHUNK는 FIFO depth와 backpressure 조건을 같이 봐야 합니다.
